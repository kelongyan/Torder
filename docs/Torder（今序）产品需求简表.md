# Torder（今序）产品需求简表

## 1. 产品定义

- 产品名称：Torder（今序）
- 产品类型：Windows 个人待办桌面软件
- 产品定位：轻量、本地优先、快速记录和处理日常任务
- 核心价值：打开快、记录快、查找快、状态清楚、数据可靠
- 首发平台：Windows
- 包管理器：统一使用 pnpm，不使用 npm

## 2. 目标用户与场景

### 目标用户

- 长时间使用电脑工作或学习的个人用户
- 需要随手记录零散任务，但不想使用复杂项目管理软件的用户
- 重视本地数据和低网络依赖的用户

### 主要场景

1. 用户启动软件，在今日页按 Enter 快速记录任务。
2. 用户集中查看过期和今天到期的任务，并逐项完成。
3. 用户在任务详情中补充备注、截止时间、提醒、优先级、清单和标签。
4. 用户通过关键词和组合筛选找回任务。
5. 用户导出完整备份，或从合法备份恢复本地数据。

## 3. V1.0 产品边界

### 必须实现

- 任务创建、编辑、完成、取消完成和删除
- 今日、全部、已完成、过期四个智能视图
- 截止时间、提醒时间、优先级、清单和标签
- 搜索及多条件筛选
- SQLite 本地持久化和数据库迁移
- 浅色、深色、跟随系统主题
- JSON 导入导出
- 应用运行期间的系统提醒
- 基础托盘能力：打开主窗口、退出应用
- Windows 安装包

### 明确不做

- 账号、云同步、移动端和团队协作
- AI 任务拆解、插件系统和复杂统计
- 看板、甘特图和复杂项目管理
- 自然语言日期解析
- 重复任务、日历视图和拖拽排序
- 全局快捷添加和托盘快速新建
- 回收站及软删除恢复界面
- 批量清理已完成任务

## 4. 信息架构

### 智能视图

- 今日：过期未完成任务、今天到期任务；可折叠显示今天完成的任务
- 全部：所有未完成且未删除的任务
- 已完成：所有已完成且未删除的任务
- 过期：截止时间早于当前时间的未完成任务

“今天”是智能视图，不作为用户清单。

### 默认清单

- 收件箱：默认清单，不指定清单时自动归入
- 工作
- 生活

### 任务详情

采用右侧抽屉，包含：标题、备注、状态、清单、标签、截止时间、提醒时间、优先级、创建时间和更新时间。

## 5. 核心业务规则

### 任务状态

- `todo`：未完成
- `done`：已完成
- `archived`：仅在数据模型预留，V1.0 不提供操作入口

### 快速创建

- 在今日视图快速创建的任务，默认截止时间为用户本地时区当天 23:59:59。
- 在具体清单中快速创建的任务，默认归入当前清单且不设置截止时间。
- 从其他入口创建且未指定清单的任务，默认归入收件箱。

### 日期与时区

- 数据库存储 RFC 3339/ISO 8601 UTC 时间，界面按 Windows 当前本地时区显示。
- 仅选择日期而未指定时间时，截止点按用户本地时区当天 23:59:59 处理。
- “今天”以用户当前本地时区的自然日为边界。
- 无截止时间的任务不因创建于今天自动进入今日视图。

### 排序

默认顺序：过期未完成、今天到期、有提醒时间、高优先级、手动排序预留值、创建时间倒序。

### 搜索与筛选

- 搜索范围：任务标题、备注、标签名称、清单名称。
- 多个筛选条件统一使用 `AND`。
- 同一筛选维度内选择多个值时使用 `OR`，例如“工作或生活”清单。

### 删除

- 界面删除前二次确认。
- 数据层使用软删除，普通视图和搜索均排除已删除任务。
- V1.0 不提供恢复入口。

### 导入导出

- 导出包含任务、清单、标签、关联关系和设置，并携带格式版本。
- 导入前完成结构及引用完整性校验。
- V1.0 采用“确认后整体恢复”模式，不做记录级合并。
- 导入必须在事务中完成；失败时保留现有数据不变。

### 提醒与托盘

- V1.0 只保证应用进程运行期间触发提醒。
- 提醒成功触发后写入状态，避免同一提醒重复弹出。
- 基础托盘只包含打开主窗口和退出应用。

## 6. 界面与交互原则

- 启动后直接进入今日视图，不显示营销首页。
- 桌面端采用左侧导航、中间任务列表、右侧详情抽屉的结构。
- 快速输入框始终位于任务列表顶部，按 Enter 创建。
- 点击复选框完成任务；点击任务主体打开详情；按 Esc 关闭抽屉或弹层。
- 高频操作路径不超过两步。
- 风格关键词：安静、清爽、克制、高对比、信息密度适中。
- 优先保证可读性、键盘操作、焦点状态和合理点击区域。

## 7. V1.0 验收标准

- Windows 应用可以安装、启动、退出和卸载。
- 用户能完整执行任务的创建、查看、编辑、完成和删除流程。
- 四个智能视图数据符合冻结规则。
- 搜索、筛选、清单和标签关联结果正确。
- 应用重启后数据和设置不丢失。
- 合法备份能完整恢复；非法文件不会破坏现有数据。
- 应用运行期间可触发一次性系统提醒。
- 主题切换立即生效并在重启后保留。
- 主流程无阻塞崩溃、数据丢失或明显布局错乱。
- 1000 条任务的搜索和筛选在目标机器上保持可用和流畅。

## 8. 需求冻结结论

阶段 0 完成后，新增需求默认进入 V1.1 需求池，不直接插入 V1.0。若需求影响数据模型、安全或主流程，必须先记录变更原因、影响阶段和处理结论。
