# Torder（今序）MVP 功能清单

## 1. P0：发布阻塞功能

### 桌面应用基础

- [ ] 使用 Tauri 2 + React + TypeScript + Vite 初始化工程
- [ ] 统一使用 pnpm，不使用 npm
- [ ] Windows 开发模式可启动
- [ ] 应用名称和窗口标题为 Torder（今序）
- [ ] 生成可安装的 Windows 安装包

### 数据层

- [ ] 初始化 SQLite 数据库和默认清单
- [ ] 建立可追踪的数据库迁移机制
- [ ] 实现 tasks、lists、tags、task_tags、settings 表
- [ ] 所有数据库访问统一经过 Repository
- [ ] 写操作使用事务，删除使用软删除
- [ ] 应用重启后数据完整恢复

### 任务闭环

- [ ] 快速创建任务
- [ ] 查看和编辑任务详情
- [ ] 设置备注、截止时间、提醒时间和优先级
- [ ] 设置所属清单和标签
- [ ] 标记完成及取消完成
- [ ] 二次确认后删除任务
- [ ] 正确展示今日、全部、已完成和过期视图
- [ ] 实现冻结版默认排序规则

### 查找与整理

- [ ] 搜索标题、备注、标签和清单名称
- [ ] 按状态、日期、优先级、清单和标签筛选
- [ ] 多维筛选条件按 `AND` 组合
- [ ] 支持清单和标签的基础管理
- [ ] 1000 条任务搜索筛选保持流畅

### 设置与数据控制

- [ ] 支持跟随系统、浅色和深色主题
- [ ] 设置修改后持久保存
- [ ] 导出带格式版本的完整 JSON 备份
- [ ] 导入前校验，确认后事务性整体恢复
- [ ] 非法导入不改变现有数据
- [ ] 展示应用版本和数据存储位置

### 桌面能力

- [ ] 应用运行期间扫描到期提醒
- [ ] 通过 Windows 系统通知提醒
- [ ] 同一提醒不会重复触发
- [ ] 托盘可打开主窗口和退出应用
- [ ] 支持 Ctrl+N、Ctrl+F、Ctrl+, 和 Esc

### 质量底线

- [ ] 主流程测试通过
- [ ] 空状态、加载状态和错误状态清晰
- [ ] 键盘焦点和常用控件具备基础无障碍能力
- [ ] 无 P0 崩溃、数据丢失和安装启动问题
- [ ] 导入导出和数据库升级经过恢复验证

## 2. P1：允许在 V1.0 收口期取舍

- [ ] 点击系统通知后定位对应任务
- [ ] 开机启动
- [ ] 今日已完成任务折叠区
- [ ] 同一维度多选筛选
- [ ] 托盘显示今日未完成数量
- [ ] 过期提醒的补发提示

P1 不得阻塞 P0 主流程。若开发周期不足，可转入 V1.1，并在阶段记录中说明。

## 3. V1.1 以后

- 重复任务
- 日历视图
- 拖拽排序
- 全局快捷添加
- 托盘快速新建任务
- Markdown 备注预览

## 4. 范围变更规则

任何新增功能必须先回答：

1. 是否直接影响个人待办核心闭环？
2. 不加入 V1.0 是否会阻塞安装、数据安全或主流程？
3. 是否会改变数据库模型、导入导出格式或阶段排期？

只有明确阻塞 P0 的需求才能进入当前版本，其余进入后续需求池。
